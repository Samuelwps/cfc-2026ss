# 🚀 GUIA COMPLETO DE SETUP E DEPLOY

## PARTE 1: SETUP LOCAL

### 1.1 Pré-requisitos
- Node.js 16+ instalado
- npm ou yarn
- Git configurado

### 1.2 Clonar e Instalar
```bash
cd "c:\Users\desktop\Documents\CFC 2026\cfc-26"
npm install
```

### 1.3 Criar arquivo .env.local
Copiar `.env.local` com as credenciais:
```env
VITE_SUPABASE_URL=sua_url_aqui
VITE_SUPABASE_ANON_KEY=sua_chave_aqui
```

### 1.4 Testar Localmente
```bash
npm run dev
```
Acessar: http://localhost:5173

### 1.5 Build Local
```bash
npm run build
npm preview
```

## PARTE 2: SETUP SUPABASE (Tiragem de Faltas)

### 2.1 Criar Projeto
1. Ir para https://supabase.com
2. Sign up / Login
3. Criar novo projeto
4. Aguardar criar database

### 2.2 Configurar Tabelas
Na aba "SQL Editor", rodar:

```sql
-- Criar tabela students
CREATE TABLE students (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  numero INTEGER UNIQUE NOT NULL,
  nome TEXT NOT NULL,
  status TEXT NOT NULL DEFAULT 'CFC',
  created_at TIMESTAMP WITH TIME ZONE DEFAULT now()
);

-- Criar tabela resets
CREATE TABLE resets (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  reset_date DATE UNIQUE NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT now()
);

-- Índices
CREATE INDEX idx_students_numero ON students(numero);
CREATE INDEX idx_students_status ON students(status);
```

## PARTE 3: SETUP VERCEL

### 3.1 Criar Conta
1. Ir para https://vercel.com
2. Sign up com GitHub
3. Conectar repositório

### 3.2 Adicionar Vercel Postgres
1. Dashboard → Storage
2. "Create Database" → Postgres
3. Conectar ao projeto

### 3.3 Obter Connection String
Após criar:
1. Ir para database settings
2. Copiar "POSTGRES_URL_NON_POOLING" ou ".env.local"

### 4.1 Acessar Settings

```

POSTGRES_PASSWORD = ...

```

## PARTE 7: TROUBLESHOOTING

### Problema: Banco não conecta
**Solução**: Verificar POSTGRES_URL no .env.local

### Problema: Supabase não sincroniza
**Solução**: Verificar VITE_SUPABASE_URL e chave

### Problema: Upload de PDF falha
**Solução**: Verificar limite de tamanho (5MB max)

### Problema: API retorna 500
**Solução**: Verificar logs em Vercel Dashboard → Functions

### Problema: Página em branco
**Solução**: 
1. Verificar Console (F12)
2. Verificar Network tab
3. Verificar Vercel logs

## PARTE 9: SEGURANÇA

### RLS Policies (Supabase)
Tabela students está pública para leitura e escrita (por design, acesso público).

Para aumentar segurança:
```sql
ALTER TABLE students ENABLE ROW LEVEL SECURITY;

-- Leitura pública
CREATE POLICY "Public read" ON students
  FOR SELECT USING (true);

-- Escrita pública (ou restrita)
CREATE POLICY "Public write" ON students
  FOR UPDATE USING (true);
```

### CORS
Vercel auto-configura CORS. Nenhuma config adicional necessária.
